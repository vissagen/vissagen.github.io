Portfolio Transfer Project

Home Portfolio Transfer

This project involved a complex data migration initiative to transfer an insurance portfolio from one insurer to another. The primary objective was to ensure a smooth transition, minimize disruptions to policyholders, and align the portfolio with the new insurer's systems and processes.

Agile Project Management

Employed Agile methodologies to manage the project, conducting regular sprint planning and daily stand-ups to ensure timely completion of deliverables.

Collaborated with cross-functional teams, including database administrators, developers, and stakeholders, to iteratively address challenges and refine the migration process.

Data Transfer Management

Migrated active policies to the new insurer, ensuring terminated and matured policies were excluded to maintain data integrity.

Identified and mapped all relevant database tables, including undocumented legacy tables used by features over a decade old, ensuring complete and accurate data extraction.

Database and Systems Analysis

Performed a thorough gap analysis of both insurers’ systems to identify discrepancies and modernize the legacy system.

Upgraded the legacy system to ensure compatibility with the receiving company's latest software, facilitating a smooth transfer.

System Customization and Branding Updates

Customized the system to reflect the new insurer's branding, including

- Updating email signatures, sender names, and email addresses for all system-generated emails.

- Rebranding documents by updating logos, signatures, and company names on all generated outputs.

Testing and Quality Assurance

Developed comprehensive test plans and test cases to ensure data accuracy and system functionality.

Performed regression testing to verify that changes did not introduce new bugs.

Collaborated with stakeholders to conduct user acceptance testing (UAT) to ensure the system met their requirements.

Legacy System Documentation Gaps

Faced difficulties with outdated or missing documentation for legacy database tables and system features.

Overcame these challenges by reverse-engineering the database structure and working closely with stakeholders to confirm accurate data sources for migration.

Software Compatibility Issues

Resolved discrepancies between the older and newer systems by upgrading the legacy software, ensuring full compatibility with the receiving insurer’s platform and avoiding any operational disruptions.

Outcome

Successfully transferred the active portfolio with zero data loss or errors.

Enhanced the receiving insurer’s branding across all system-generated emails and documents, streamlining operations and ensuring brand consistency.

Delivered the project on time and within scope using Agile methods, promoting flexibility and responsiveness to emerging challenges.

Tools & Technologies Used

SQL (for database querying and data extraction)

Confluence and JIRA (for project documentation and task tracking)

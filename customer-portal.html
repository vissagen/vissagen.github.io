Customer Portal Project

Home Customer Portal

This project involved developing a customer portal to enhance user experience by providing seamless access to policy details, improving customer satisfaction, and reducing the volume of support requests.

Requirements Gathering

Collaborated with various teams to understand common customer support requests and identify features that would add the most value to the portal.

Feature Implementation

Enable users to access critical information such as policy details, premium payment history, and investment performance.

Implement self-service features to empower users to manage their policies independently.

Data Integration

Develop an API to securely extract and transmit required data from the existing system to the customer portal.

Implement data synchronization processes to ensure data consistency and real-time updates.

Multi-Factor Authentication (MFA)

Implement robust security measures, including multi-factor authentication (MFA) using SMS and email OTP, to safeguard sensitive customer information.

Infrastructure and Hardware Upgrades

Led the installation of new hardware components necessary for the portalâ€™s operation, ensuring robust performance and scalability.

Security Testing

Conducted penetration testing (pentest) of the application to ensure compliance with data security standards and safeguard sensitive customer information.

Cross-Team Communication

Encountered communication challenges as the development team, while proficient in understanding English, faced difficulty speaking it fluently. Overcame this by establishing clear, detailed documentation and holding regular meetings to ensure accurate communication and alignment on project goals.

Data Extraction Challenges

Addressed data extraction issues by providing the development team with detailed insights into database structures and identifying the necessary data sources for integration.

Outcome

Successfully launched the customer portal, resulting in improved customer satisfaction due to easier access to policy and financial information.

Achieved a significant reduction in support requests as customers could now manage their own information through self-service features.

Ensured the portal met stringent security standards through comprehensive pentesting and secure data handling practices.

SQL (for database management and data extraction)

Microsoft Teams and SharePoint (for collaboration)

Multi-factor authentication (SMS and Email OTP)
